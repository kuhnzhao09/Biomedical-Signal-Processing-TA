# New Materials Intake Guide

这份规范用于后续给 `Biomedical Signal Processing TA` 持续添加新课件、讲义、实验指导书和数据说明，并尽量保证这些资料能被智能体稳定检索和回答。

## 1. 目标

新增资料接入后，应满足以下目标：

- 能被检索层读取
- 能按章节和主题命中
- 能优先覆盖旧的通用资料
- 能在回答中被引用为参考资料
- 不需要重新训练模型

## 2. 推荐目录结构

建议在仓库外或本地私有目录中维护完整教学资料，再将允许公开的部分择优同步到仓库。

如果只看知识库组织，推荐按下面结构整理：

```text
materials/
  course-slides/
    chapter-01-introduction/
    chapter-02-sampling-and-filtering/
    chapter-03-ecg/
    chapter-04-emg/
    chapter-05-eeg/
    chapter-06-ppg-fnirs/
  lecture-notes/
  lab-guides/
  assignments/
  faq/
  datasets/
  rubrics/
```

如果你暂时不想公开上传这些资料，也可以在本地沿用同样结构，后续由检索脚本直接读本地目录。

## 3. 最推荐的资料类型

优先级从高到低建议如下：

1. 教师自编课件 PDF
2. 教师讲义或章节笔记 PDF / Markdown
3. 实验指导书 PDF / Markdown
4. 作业说明与评分标准
5. 常见问答 FAQ
6. 外部公开教程或综述

原因很直接：越贴近你的课程，智能体越像“你的助教”，而不是通用问答机器人。

## 4. 推荐文件格式

最适合接入的格式：

- `pdf`
- `md`
- `txt`
- `html`

可接入但通常需要额外处理的格式：

- `pptx`
- `docx`
- 图片型或扫描型 PDF

说明：

- `PDF` 是目前最稳妥的课件格式。
- 如果是 `PPTX`，建议同时导出一份 `PDF` 版本再入库。
- 如果是扫描版 PDF，必须先做 OCR，否则检索质量会明显下降。

## 5. 单个文件的命名规范

建议文件名直接体现章节和主题，避免 `最终版`、`修改版2` 这类命名。

推荐形式：

```text
ch03_ecg_qrs_detection.pdf
ch04_emg_preprocessing.pdf
lab02_ecg_filtering_and_r_peak_detection.pdf
faq_assignment1_sampling_aliasing.md
rubric_lab1_ecg_report.md
```

这样做的好处：

- 文件本身就携带主题信息
- 便于检索结果显示给学生
- 更容易做自动分组和优先级判断

## 6. 新资料接入的最小流程

每次新增资料，至少做下面 4 步：

1. 放入正确目录
2. 统一命名
3. 确认文件可解析
4. 更新检索优先级配置

对应到你当前项目，最小操作建议如下：

1. 将新资料放入约定目录
2. 如果是 `pptx`，先导出为 `pdf`
3. 如果是扫描 PDF，先 OCR
4. 将该资料加入 `retrieval-priority-v1.json` 对应分组
5. 如果是 Cloud 版，重新部署或重新加载应用
6. 如果是本地 `Chroma` 版，重建索引

## 7. 如何让新资料优先于旧资料

这是最关键的一步。

新课件接进来后，不是一定会自动排到最前面。要想让它优先回答，需要至少做下面一项：

- 将新资料加入更高优先级的 `source_group`
- 将旧的通用资料降级为补充资料
- 在特定路由中优先命中新资料目录

例如：

- 新的 `ECG` 课件应优先放到 `ecg_core`
- 新的 `EEG` 实验指导书应优先放到 `eeg_core`
- 通用公开教程保留在背景组，不要与课程核心资料同级

## 8. 针对不同资料类型的接入建议

### 8.1 课件

适合回答：

- 概念解释
- 课堂重点
- 章节总结
- 考点梳理

建议：

- 每章一份 PDF
- 首页标明章节号和主题
- 标题尽量明确，例如“ECG 信号预处理与 QRS 检测”

### 8.2 实验指导书

适合回答：

- 实验步骤
- 参数选择
- MATLAB / Python 实现思路
- 结果解释

建议：

- 单独归档到 `lab-guides/`
- 文件名带 `lab01`、`lab02` 等编号
- 保留实验目的、流程、结果分析几个固定模块

### 8.3 作业说明

适合回答：

- 解题提示
- 任务理解
- 注意事项

建议：

- 单独归档到 `assignments/`
- 保留“题目要求”和“评分点”两部分
- 学生模式下只允许提示，不允许直接生成提交级答案

### 8.4 FAQ

适合回答：

- 高频重复问题
- 术语混淆
- 软件使用问题

建议：

- 用 `md` 最方便维护
- 一问一答，单条不要太长

## 9. 哪些资料不建议直接入库

- 来源不明的整本教材 PDF
- 版权状态不明确的商业教材扫描件
- 纯图片、无 OCR 的扫描文件
- 内容严重重复、版本混乱的旧文件
- 只有结论没有过程的零散截图

这些资料即使能读进去，也会拉低检索稳定性。

## 10. 版本更新建议

每次课程更新后，推荐这样处理：

- 新版本资料保留清晰版本号或学期标识
- 旧版资料不要直接混在同一层级
- 当前学期资料作为核心组
- 往年资料降为补充组

推荐命名示例：

```text
2026s_ch03_ecg_qrs_detection.pdf
2026s_lab02_ecg_preprocessing.pdf
2025f_ch03_ecg_qrs_detection.pdf
```

## 11. 与当前项目的具体对应

你当前仓库里，后续最需要维护的是：

- `agent-template/kb-v1/retrieval-priority-v1.json`
- `agent-template/kb-v1/retrieval-priority-v1.yaml`
- 如果是本地工程版：`agent-template/kb-v1/langchain_chroma_rag.py` 构建后的索引
- 如果是 Cloud 轻量版：资料文件本身是否被纳入检索目录

简单理解：

- `资料文件` 决定“能不能答”
- `优先级配置` 决定“先用谁答”
- `学生/教师模式` 决定“怎么答”

## 12. 推荐的半自动更新流程

后续最实用的流程是：

1. 你把新资料放进固定目录
2. 脚本自动扫描新增文件
3. 自动提取文本
4. 自动按文件名和目录打标签
5. 自动加入候选资料清单
6. 由你人工确认后更新优先级配置
7. 自动重建本地索引或重新部署 Cloud

也就是说，最好的做法不是完全自动，而是“自动提取 + 人工确认优先级”。

## 13. 你后续新增资料时的最低执行标准

建议你后面每次上传新资料，都至少满足这 5 条：

- 文件名明确
- 格式可解析
- 放在正确目录
- 指定所属主题组
- 明确是否高于旧资料优先级

只要做到这几条，智能体就能比较稳定地提取知识点并回答。

## 14. 下一步可继续做的事情

如果你后续要把这个流程再推进一层，建议优先做这两个：

1. 新增一个“资料扫描与入库脚本”
2. 新增一个“资料优先级维护表”

前者解决“怎么自动发现新资料”，后者解决“新资料进来以后谁优先回答”。
