# Knowledge Base Template

Use this structure to turn the current workspace into a practical knowledge base for the teaching agent.

## Recommended Folders

- `materials/books/`
  Put textbooks, monographs, and broad reference PDFs here.
- `materials/papers/`
  Put topic-specific papers or methods papers here.
- `materials/course-outline/`
  Put course schedule, chapter list, and topic priorities here.
- `materials/labs/`
  Put lab tasks, datasets, report templates, and sample outputs here.
- `materials/code/matlab/`
  Put MATLAB demos, scripts, and helper functions here.
- `materials/code/python/`
  Put Python demos, notebooks, and helper scripts here.
- `materials/q-and-a/`
  Put common student questions and your preferred answers here.
- `materials/rubrics/`
  Put grading criteria, report scoring guides, and assignment boundaries here.

## What You Can Do Right Now

With only general books and papers available, the agent can already support:

- concept explanation,
- generic lab workflow guidance,
- MATLAB and Python example snippets,
- exercise generation,
- code review for common signal-processing mistakes.

## What Will Improve Later

When you add course-specific files, the agent will become better at:

- matching your chapter order,
- reusing your notation,
- following your lab conventions,
- aligning with your assessment style,
- answering from your own course material instead of generic references.

## Minimum Useful Additions

Add these first when available:

1. A chapter list for the undergraduate course.
2. One MATLAB example for ECG or EMG.
3. One Python example for plotting or filtering.
4. One lab instruction sheet.
5. One grading rule or homework boundary note.
